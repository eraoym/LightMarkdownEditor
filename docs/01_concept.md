# 01 コンセプト・要件定義（MVP）

> 最終更新: 2026-03-16

---

## コアコンセプト

- **「今すぐ書いて、すぐ見れる」**

ファイルを開く、または何も考えずにウィンドウを起動してすぐ書き始められる。
書いた内容が隣にリアルタイムでレンダリングされる。それだけでいい。

---

## MVPのスコープ

### やること

| # | 機能 | 説明 |
| - | ---- | --- |
| 1 | 新規作成 | 起動時に空の編集画面を表示する |
| 2 | ファイルを開く | OS のファイルダイアログで `.md` ファイルを選択して読み込む |
| 3 | リアルタイムプレビュー | EditMode / PreviewMode をヘッダーのトグルで切り替え。編集中はプレビューが即時反映される |
| 4 | 上書き保存 | `Ctrl+S` で現在開いているファイルに保存する |
| 5 | 名前を付けて保存 | 新規ファイルの初回保存時はダイアログで保存先を指定する |

### やらないこと（MVP以降）

- シンタックスハイライト（コードブロック内の色付け）
- 設定画面（フォントサイズ変更、テーマ切替など）
- 自動保存

---

## ユーザーフロー

```text
起動
 │
 ├─ [前回ファイルなし] → 空の編集画面
 │
 └─ （将来）前回ファイルを復元

編集中
 │
 ├─ キー入力 → プレビューモードに切り替えると即時反映を確認できる
 │
 ├─ ヘッダー「編集」「プレビュー」ボタン → モード切替（全画面表示）
 │
 ├─ Ctrl+S
 │    ├─ [保存済みファイル] → 上書き保存
 │    └─ [未保存の新規] → 名前を付けて保存ダイアログ
 │
 └─ 「開く」ボタン → ファイルダイアログ → 読み込み → 編集画面に反映
```

---

## 成功基準

- [ ] ウィンドウを起動してから **3秒以内** に書き始められる
- [ ] キー入力に対してプレビューが **目に見える遅延なし** で更新される
- [ ] `Ctrl+S` 一発で保存できる
- [ ] `.md` ファイルをダブルクリックで開ける（将来対応、MVP外）
